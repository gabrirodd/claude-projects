# Corpus Argument Analysis — Claude Code Instructions

## What this project does

Analyses the argument structure, rhetorical logic, and epistemic mode of 50 academic
papers from a shared intellectual tradition. Produces a typology of argument patterns
at multiple levels of granularity, with canonical paper examples for each type.

No external API calls. All analysis is performed by Claude Code reading paper texts
directly. Python scripts handle only file I/O and schema validation.

## The extraction schema

Before analysing any paper, read `scripts/profile_schema.json` carefully.
Every field must be filled. The most important fields are:

**`move_sequence`** — ordered list of functional move labels. This is what
enables cross-paper comparison. Use ONLY labels from the schema vocabulary.
Do not invent new labels. If a move is present but doesn't fit any label,
use the closest one and note the deviation in `confidence_notes`.

**`epistemic_mode.primary`** — the most important classification dimension.
Use ONLY the values listed in the schema. Key distinctions:
- `fully_normative` = argument moves entirely within conceptual/normative space,
  no empirical evidence or interpretive case
- `normative_with_interpretive_case` = film, literary text, historical event,
  philosophical text used as a lens — THIS is what Ungureanu does
- `normative_with_qualitative_empirical` = interviews, ethnography, discourse
  analysis deployed in service of a normative argument
- `normative_with_quantitative_empirical` = statistical data supports normative claim
- `purely_interpretive_hermeneutic` = close reading or genealogy without explicit
  normative conclusion
- `primarily_empirical_qualitative` / `_quantitative` = empirical argument primary,
  normative element secondary or absent

**`empirical_evidence_role`** — even when empirical material is present, ask:
what work is it doing? `constitutive` means the argument cannot stand without it.
`illustrative` means it decorates a point already established conceptually.
`supportive` means it strengthens but doesn't constitute. This distinction is
crucial for the typology.

## Step-by-step workflow

### Step 0 — Setup (once)

All scripts must be run from the project root (the directory containing this file).

```bash
python -m venv .venv
source .venv/bin/activate
pip install pdfplumber
python scripts/extract.py
```
Confirm that `corpus/texts/` now contains one `.txt` per PDF. This directory is created automatically by `extract.py` — it does not exist until then.

### Step 1 — Analyse papers (the main work)

For EACH paper in corpus/texts/, do the following IN ORDER:

1. Run: `python scripts/analyse_paper.py --paper {paper_stem}`
   This prints the full paper text to your context.

2. Read the paper text carefully. You are reading as a rhetorician, not a content
   reader. For each major section, ask: what argumentative FUNCTION does this serve?
   Not what does it say, but what move is it making?

3. Read `scripts/profile_schema.json` to confirm field requirements.

4. Write the profile to `corpus/profiles/{paper_stem}.json`.
   Follow the schema exactly. Fill every field.

5. Validate: `python scripts/analyse_paper.py --paper {paper_stem} --validate-only`
   If validation fails, fix the profile and re-validate.

6. After every 10 papers, pause and note any patterns already visible across profiles.
   Add these observations to a file `corpus/emerging_patterns.md`.
   These early observations will improve analysis of subsequent papers.

**Reading heuristics for functional move identification:**

- Opening paragraph(s) = almost always `puzzle_diagnosis` or `empirical_presentation`
- Section explicitly titled "literature review" or surveying positions
  = `rival_mapping`, possibly followed by `immanent_critique`
- "In contrast to X, I argue..." = `framework_proposal`
- Introduction of a film/case/event = `case_introduction`
- Close reading or detailed interpretation = `case_development` or `case_interpretation`
- "However, one might object that..." = `counter_objection_response`
- "This analysis has limits..." or "I have argued X but..." = `meta_reflexive_qualification`
- Final section drawing conclusions = `conclusion_strong` or `conclusion_open`

**Epistemic mode heuristics:**

Ask these questions in sequence:
1. Does the paper use any empirical data (surveys, interviews, archives, statistics)?
   If yes → not `fully_normative`. Determine whether quant or qual.
2. Does the paper analyse a cultural object (film, novel, legal case, speech)?
   If yes → likely `normative_with_interpretive_case` (if normative stake is high)
   or `purely_interpretive_hermeneutic` (if no normative conclusion drawn).
3. Is the empirical/interpretive material constitutive or merely illustrative?
   Constitutive = the typology shifts toward empirical primary.
   Illustrative = the typology stays normative_with_X.
4. Is there a normative conclusion (a claim about what ought to be, what is
   pathological, what democracy requires)? If absent → purely_empirical or
   purely_interpretive.

### Step 2 — Aggregate

```bash
python scripts/cluster.py
```

This writes `corpus/all_profiles.json` and `outputs/epistemic_map.csv`.
Read the console report — it shows the distribution of epistemic modes and
flags any profiles with unusual characteristics.

### Step 3 — Typology synthesis (the core analytical step)

Read the full `corpus/all_profiles.json`.

Your task: identify recurring argument grammars across the 50 profiles.
A grammar is a recurring pattern of move_sequence + epistemic_mode + case_type
+ normative_stake that appears in multiple papers.

**Procedure:**

1. Group papers by `move_sequence` similarity first (ignore content, focus on
   functional structure). Papers with identical or near-identical sequences
   form candidate clusters.

2. Within each candidate cluster, check whether `epistemic_mode` and `case_type`
   are also consistent. If yes, you have a strong typology.

3. Name each typology descriptively (not alphabetically). The name should capture
   the LOGIC, not the topic. E.g.:
   - "Immanent critique via cultural parable" (not "Type A")
   - "Reconstructive normativism without case"
   - "Empirically grounded critical diagnosis"

4. For each typology, identify:
   - The canonical example (paper that best fits)
   - A near-canonical example (fits well but with one deviation)
   - The typical move sequence
   - The typical epistemic mode
   - What distinguishes this type from adjacent types

5. Document papers that resist clean classification in `deviation_notes.md`.
   These are often theoretically interesting — they may represent hybrid forms
   or transitional argument logics.

### Step 4 — Write the output document

Write `outputs/typology_report.md` following this exact structure:

## Quality standards

**Never skip the validation step.** An invalid profile will corrupt the
clustering and typology steps downstream.

**Never invent move labels.** Use only the vocabulary in the schema.
If a paper makes a move not in the vocabulary, use the closest label
and note the gap in `confidence_notes`. After 10 papers, report whether
a vocabulary gap seems systematic — it may indicate the schema needs extending.

**The functional reading is the hard part.** The most common mistake is
conflating content with function. "Section 3 argues that Habermas's theory
of communicative action cannot account for founding violence" is a content
summary. "Section 3 performs immanent_critique of the dominant progressive
position" is a functional label. Always ask: what is this section DOING,
not what is it SAYING.

**Epistemic mode requires honest assessment.** If a paper uses one statistical
table to illustrate a fully normative argument, that is `normative_with_quantitative_empirical`
with `empirical_evidence_role: illustrative` — not `primarily_empirical_quantitative`.
The function, not the presence, of empirical material determines the classification.

**Pay special attention to papers that combine empirical and normative moves.**
These are usually the most interesting — note explicitly how the paper negotiates
the transition from empirical findings to normative conclusions (if it does),
or whether it leaves that transition unjustified (which is itself a finding).
